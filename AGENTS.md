# AGENTS.md — дом (средняя строгость)

Ты работаешь в доме — конечном продукте со своим релизным циклом. Радиус поражения — один дом.

## Правила дома

- **Гейт перед завершением любой задачи — механический: pre-push хук skipi-guard + check `guard` в CI на PR.** Красный гейт = задача не завершена: чини или фиксируй инцидент, но не закрывай и не обходи. Локальной цели `make verify-fast` в доме пока нет — она появится вместе с пилотным CI (skipi-ops BACKLOG п.2) и тогда станет обязательной.
- **contracts/ и skipi-verify не трогать.** Контракты и гейт меняет только человек через ревью; нужно изменение — handoff в штаб.
- **Флот systemd-юнитов (broker-*, trader-*, consultant-*, ag*) — не останавливать, не читать без задачи, не трогать:** это работающая доходная автоматика, другой мир.

## Общий протокол (одинаков во всех репо Skipi)

- Одна задача = один worktree = одна сессия. Задача закрыта — сессия закрывается; новая задача = свежая сессия.
- Минимальный дифф: меняй только то, что требует задача.
- Bugfix = сначала failing-test, воспроизводящий баг → фикс → зелёный прогон. Фикс без failing-test'а не принимается.
- После работы: commit + push.
- `--no-verify` запрещён.
- systemd, деплой, прод-сервер и флот ботов — вне скоупа; нужно — handoff в штаб (`skipi-ops/handoffs/`).
- Инцидент (регрессия, потеря данных) → SKI-INC-NNN в `skipi-ops/incidents/`.
- Сессия запускается из папки этого репо (`cd <репо> && claude`), не из корня. Если этого файла нет в списке `/memory` — сессия поднята неправильно: остановись и сообщи человеку.
- Обмен с другими агентами — через артефакты на диске (коммиты, handoffs, verify-отчёты), не копипастом между окнами.
